# Reproducibility

This section documents everything needed to re-run the Framed Listening NLP
analysis pipelines from scratch. All data and code are openly available at
<https://github.com/HazelvdW/context-framed-listening>.

---

## Software versions

| Software | Minimum version | Developed on |
|---|---|---|
| Python | 3.10 | 3.12.3 |
| R | 4.2 | 4.4.x |
| Quarto | 1.4 | 1.5.x |
| reticulate (R) | 1.35 | 1.38.x |

---

## Pipeline documents

| File | Purpose | Key packages |
|---|---|---|
| `NLP-text-prep.qmd` | Spell-checking, stop word removal, lemmatisation, combMIMC aggregation | pandas, numpy, nltk, pyspellchecker |
| `NLP-TFIDF-analysis.qmd` | TF-IDF semantic similarity (combMIMC level) | scikit-learn, scipy, wordcloud |
| `NLP-SEmb-analysis.qmd` | Sentence-embedding similarity (individual level) | sentence-transformers, torch, scikit-learn |
| `NLP-model-comparison.qmd` | Cross-model top/bottom pair comparison (all models × both levels) | all of the above + gensim |

Run in order: text-prep → TFIDF → SEmb → model-comparison.

## Python environment

### Option A — pip (recommended)

```bash
# 1. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate        # macOS / Linux
.venv\Scripts\activate           # Windows

# 2. Install minimum-version dependencies
pip install -r requirements.txt

# 3. Optional: pin to exact development versions
pip install -r requirements.txt --constraint constraints.txt
```

### Option B — conda

```bash
conda create -n framed-listening python=3.12
conda activate framed-listening
pip install -r requirements.txt
```

### GPU / CPU note for torch

`requirements.txt` lists `torch` without a CUDA specifier, so pip installs the
CPU build by default. This is sufficient for all pipelines (inference only,
no training). For faster sentence-embedding generation on a CUDA GPU:

```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

### NLTK data download

`NLP-text-prep.qmd` downloads NLTK corpora at runtime via `nltk.download()`.
To pre-download manually:

```python
import nltk
nltk.download('stopwords')
nltk.download('punkt_tab')
nltk.download('wordnet')
nltk.download('averaged_perceptron_tagger_eng')
```

### Word2Vec model download

`NLP-model-comparison.qmd` downloads the pre-trained Google News Word2Vec vectors
(~1.6 GB) on first run via gensim and caches them locally. Subsequent runs load
from cache. No manual action required.

## R environment

R dependencies are managed with [`renv`](https://rstudio.github.io/renv/).

```r
# Restore exact package versions from lockfile
install.packages("renv")
renv::restore()
```

If you get an error about `knitr` or `rmarkdown` not being found (common when
renv is isolating the project library), install them explicitly then re-snapshot:

```r
renv::install(c("knitr", "rmarkdown", "reticulate"))
renv::snapshot()
```

If setting up fresh (no `renv.lock` yet):

```r
renv::init()
renv::install(c("knitr", "rmarkdown", "reticulate"))
renv::snapshot()
```

Point reticulate at the Python virtual environment created above:

```r
reticulate::use_virtualenv(".venv", required = TRUE)
# or for conda:
reticulate::use_condaenv("framed-listening", required = TRUE)
```

## Rendering the analyses

```bash
# Render individually
quarto render NLP-text-prep.qmd
quarto render NLP-TFIDF-analysis.qmd
quarto render NLP-SEmb-analysis.qmd
quarto render NLP-model-comparison.qmd

# Or all at once (requires _quarto.yml)
quarto render
```

### Embedding cache

`NLP-SEmb-analysis.qmd` and `NLP-model-comparison.qmd` cache sentence embeddings
to `.npy` files on first run. To force re-encoding, delete the relevant cache:

```bash
# SEmb pipeline
rm NLP_outputs/BERT_individual/embeddings_<model-slug>.npy

# Model comparison pipeline
rm NLP_outputs/model_comparison/embeddings_A_*.npy
rm NLP_outputs/model_comparison/embeddings_B_*.npy
```

## File structure

```
context-framed-listening/
├── NLP-text-prep.qmd                # Pipeline 1: preprocessing + combMIMC aggregation
├── NLP-TFIDF-analysis.qmd          # Pipeline 2: TF-IDF, combMIMC level
├── NLP-SEmb-analysis.qmd           # Pipeline 3: sentence embeddings, individual level
├── NLP-model-comparison.qmd        # Pipeline 4: cross-model comparison
├── requirements.txt                 # Python minimum-version dependencies
├── constraints.txt                  # Python exact-version pins
├── DESCRIPTION                      # R package metadata (used by renv)
├── renv.lock                        # R exact-version lockfile (generated by renv)
└── NLP_outputs/
    ├── dataMIMC_preprocessed.csv    # Individual MIMC responses (all preprocessing levels)
    ├── combMIMC_lvl1a.csv           # Aggregated MIMC, light preprocessing + text_original
    ├── combMIMC_lvl1b.csv           # Aggregated MIMC, light preprocessing (thought words kept)
    ├── combMIMC_lvl2a.csv           # Aggregated MIMC, aggressive preprocessing + text_original
    ├── combMIMC_lvl2b.csv           # Aggregated MIMC, aggressive preprocessing (thought words kept)
    ├── TFIDF/                       # TF-IDF outputs
    ├── BERT_individual/             # Sentence-embedding outputs + embedding cache
    └── model_comparison/            # Cross-model comparison outputs + embedding caches
```
